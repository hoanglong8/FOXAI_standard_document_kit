Nguyên tắc kiến trúc cốt lõi
🛡️

Privacy-by-Default

Mọi cài đặt mặc định ưu tiên bảo vệ dữ liệu: mã hóa bật sẵn, minimal data collection, opt-in thay vì opt-out cho mọi xử lý không thiết yếu.

👁️

Explainability & XAI

Mọi quyết định AI rủi ro cao phải có khả năng giải thích (Explainable AI). Dashboard tích hợp trace log từ Input → Model → Output để phục vụ thanh tra.

🧑‍💼

Human-in-the-Loop

Mọi dashboard quản trị AI rủi ro cao bắt buộc có nút Kill-switch cho phép nhân sự được phân quyền ghi đè hoặc dừng khẩn cấp quyết định AI.

🔍

Anti-tamper Audit Trail

Toàn bộ nhật ký hệ thống được ký số (digital signature) hoặc hash chain để chống sửa xóa — sẵn sàng cung cấp cho Bộ Công an (A05) khi thanh tra.

🔴 Rủi ro cực cao
Prompt Injection Attack
Kẻ xấu chèn lệnh độc hại vào input để điều khiển AI. Xử lý bằng input sanitization + full logging.
🟠 Rủi ro cao
Data Poisoning (Đầu độc dữ liệu)
Dữ liệu huấn luyện bị nhiễm độc khiến model học sai. Ngăn chặn qua data lineage + hash verification.
🟠 Rủi ro cao
Model Drift (Trôi mô hình)
Hiệu suất model suy giảm theo thời gian do dữ liệu thực tế thay đổi. Kiểm soát qua monitoring liên tục.

Module 1 · Encryption & Key Management

Cơ chế mã hóa toàn diện: lưu trữ (AES-256-GCM), truyền tải (TLS 1.3), quản lý vòng đời khóa (Key Lifecycle Management) và ẩn danh hóa dữ liệu trước khi huấn luyện AI.

Cipher suitesTLS_AES_256_GCM_SHA384, CHACHA20_POLY1305
CertificateECDSA P-256, tự động gia hạn (ACME)
Certificate pinningHPKP / TrustKit bắt buộc mobile
Perfect Forward Secrecy✅ ECDHE key exchange
HSTSmax-age=31536000; includeSubDomains
mTLS internalService-to-service mutual TLS
Anonymization Pipeline cho AI Training

① Phát hiện PII

  • NER model: tên, CCCD, SĐT, địa chỉ, email
  • Regex patterns cho structured PII (tài khoản, MST)
  • Confidence score threshold: > 0.85 mới flag
  • Human review cho borderline cases

② Kỹ thuật ẩn danh

  • Pseudonymization: token reversible cho audit
  • Generalization: "Hà Nội" → "Miền Bắc"
  • Data masking: 0123-456-789 → 0123-xxx-xxx
  • Differential Privacy: noise injection aggregate

③ Xác minh sau xử lý

  • Re-identification risk score ≤ 0.09 (k-anonymity)
  • Hash fingerprint dataset trước/sau để audit
  • Tự động reject nếu còn PII vượt ngưỡng
  • Certificate anonymization kèm dataset

Module 2 · Centralized Logging & Anti-Tamper

Hệ thống ghi nhật ký tập trung cho 4 nhóm hoạt động, lưu trữ tối thiểu 12 tháng, chống sửa xóa bằng hash chain và digital signature — sẵn sàng cung cấp cho Bộ Công an (A05) khi thanh tra.

ELK Stack / OpenSearchSHA-256 Hash Chain WORM Storage12 tháng retention72h incident report
4 Nhóm nhật ký bắt buộc

🟣 Nhóm 1 – Access Logs

  • UserID, IP, timestamp (UTC+7), user-agent
  • Resource accessed (API endpoint, data entity)
  • HTTP status code, response time (ms)
  • Geo-location và device fingerprint
  • Failed login attempts và lockout events

🔵 Nhóm 2 – AI Activity Logs

  • Request input (hash + metadata, không lưu PII raw)
  • Model version, model ID, confidence score
  • Output summary + decision category
  • Inference latency và resource usage
  • Model drift score (so sánh với baseline)

🔴 Nhóm 3 – Security Alert Logs

  • Prompt injection attempts (flagged by guard model)
  • Data poisoning detection events
  • Anomaly detection: unusual access patterns
  • Privilege escalation attempts
  • DLP (Data Loss Prevention) trigger events

🟢 Nhóm 4 – Human Intervention Logs

  • Override / Reject AI decision (by whom, when, reason)
  • Kill-switch activation events
  • Manual review và final decision
  • Escalation path taken (L1 → L2 → L3)
  • Audit notes và justification text
Cơ chế chống sửa xóa (Anti-Tamper)

🃏 System Card – Thẻ hệ thống AI

  • Tên, mô tả, phiên bản hệ thống AI
  • Mức độ rủi ro phân loại (High/Medium/Low)
  • Domain áp dụng và đối tượng người dùng
  • Cơ chế giám sát và human oversight hiện có
  • Giới hạn và điều kiện sử dụng an toàn
  • Cơ chế khiếu nại và phương thức khắc phục

🧠 Model Card – Thẻ mô hình AI

  • Kiến trúc model, tham số, framework sử dụng
  • Nguồn và đặc điểm dữ liệu huấn luyện
  • Phương pháp anonymization đã áp dụng
  • Hiệu suất, giới hạn và known biases
  • Lịch sử cập nhật và version history
  • Kết quả kiểm tra fairness và XAI metrics

🗺️ DFD – Sơ đồ luồng dữ liệu tự động

  • Tự động generate từ service mesh topology
  • Ghi chú loại dữ liệu trên mỗi luồng
  • Highlight điểm xử lý dữ liệu cá nhân
  • Cơ chế bảo vệ tại mỗi điểm (mã hóa, hash)
  • Cross-border data transfer markers
  • Xuất Mermaid / draw.io / Visio format
